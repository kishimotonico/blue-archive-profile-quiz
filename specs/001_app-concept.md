# ブルアカプロフクイズアプリ計画

## アプリ概要

「ブルーアーカイブ」というゲーム内に登場するキャラクターのプロフィールに関するクイズアプリです。ユーザーは、断片的なプロフィール情報（ヒント）から生徒を特定することを目的とします。

Webベースのシングルページアプリケーションです。

## 技術仕様

- TypeScript
- React v19
    - React Router
    - jotai
    - Motion
    - zod
- Tailwind CSS (optional)
- Claude Code

## ドメイン知識

- ブルーアーカイブ内で登場する「生徒」のみを取り扱う
- 生徒は「月雪ミヤコ」のように、漢字の姓、カタカナの名で構成される(一部例外あり)

## 機能仕様

### 全体

- 日替わりクイズとフリープレイの2種類の出題形式

### クイズの共通仕様

- ヒントをランダムな順番で1つずつ開示しながら、生徒の特定を目指す
- 回答はいつでも可能
- ユーザーは生徒の名前をテキストで回答する
- 回答テキストは、ひらがなでも回答可。カタカナに変換して正誤判定する
- 回答テキストは、フルネーム（姓名）でも回答可。姓名の間にスペースが含まれていてもOK
- より少ないヒントで正解した場合、より高得点を獲得
- 誤答した場合
    - 生徒を間違えて回答した場合は、その問題は0点として終了
    - typoなど、該当する生徒が存在しない場合はそのまま続行
- 画面上には生徒の立ち絵を表示する
    - ヒントの開示中は、はてなマークで覆われた状態で表示
    - 9つのヒントをすべて開示しても分からない場合、シルエット（黒塗り）で表示
    - 正解発表時に立ち絵をフェード表示

### 日替わりクイズ

- 日替わりクイズでは、1日1問を出題する
- 4:00 JSTに問題が変わる
- クイズの内容（乱数）は日付で一意に定まる
- 全ユーザーで同じクイズを共有することになる
- クイズ終了時、結果をポップアップで表示する
    - 今日のスコア, 累積挑戦回数, スコア分布(0点, 1-2点, 3-4点, 5-6点, 7-8点, 9-10点), ベストスコア
- クイズの結果はローカルストレージに保存する
- ローカルストレージに回答済みの記録がある場合、回答時の状態(表示済みのヒント、回答の正誤)を再現する
    - 未回答であれば続きから挑戦可

### フリープレイ

- フリープレイでは、都度ランダムな10問を出題する
- 10問終了後、結果を表示する
    - 合計スコア
    - 問題ごとのスコア
- 再度、別の10問に挑戦可能なUIにする

### プロフィール

ヒントに利用するプロフィールは各項目です

- フルネーム: 回答、および正解に利用
- 姓: 名前のうち、姓のみをヒントに利用
- 学園: 所属学園名 / 学年 (例: SRT特殊学園 / 1年生)
- 所属: 部活名
- 年齢: 歳
- 誕生日: 月日
- 身長: cm単位
- 趣味
- 武器名
- CV: 声優名
- 立ち絵 (PNG): 全ヒント開示後にシルエット(黒塗り)を表示。正解発表時に表示

#### 下記は使わないプロフ

- ヘイロー (PNG)
- スキル名
    - EXスキル
    - ノーマルスキル
    - サブスキル
    - パッシブスキル
